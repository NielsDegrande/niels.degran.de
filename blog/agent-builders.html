Agent builders

Published on July 22, 2025

Agent builders are popping up everywhere. They promise to let us wire up reasoning loops, memory and tools without ever opening a code editor. But not all builders are created equal, so let's group them into a few archetypes.

No-code / form-like builders

  • CustomGPT
  • Amazon Q
  • Copilot Studio
  • LangGraph Platform

No-code tools democratize AI for business and semi-technical users. They are part of the self-serve play and will only get more capable. Expect built-in reasoning loops, reflection and memory that can be enabled with a single checkbox. The real power rests in the catalog of connectors—Slack, Teams, SharePoint, Notion, Trello and anything you can browse in an MCP registry.

Shadow AI will be a major challenge: an employee spins up an agent that nobody owns after they leave. I recommend to keep the agent builder platform inside IT, add guard-rails through an LLM gateway, and establish FinOps practices for cost allocation. Security-wise you want pass-through authentication, user-level memory and the usual network controls.

Low-code workflow and app builders

  • Retool
  • n8n
  • Power Platform

Low-code has been around forever—from classic BPM suites to DataRobot for ML—and adoption has always stayed niche. GenAI has not changed that equation yet, even if influencers share eye-catching outreach workflows on social media. Generative process automation might become a killer use case, but the value is still to be proven.

For small internal apps with a handful of users I am positive. Architecture boards should openly ask: could this be a low code agent? Just avoid the fate of “a thousand PowerBIs” by tracking ownership and cost centres.

For anything large, external or business-critical one might think twice: the total cost of ownership tends to explode due to seat pricing, logic escapes into Python/TS blocks, edge cases start to pile up, and limited code reusability makes spaghetti everywhere.

Workflow-first agent builders

  • Dify.ai
  • Langflow
  • Flowise

These products wrap LangChain-like frameworks with a visual layer. You get more building blocks than Retool or n8n, but they lag in enterprise polish and UI automation.

Embedded builders

  • Salesforce Agentforce
  • SAP Joule Agents
  • ServiceNow AI Agent Studio

Large platforms are baking agent builders directly into their ecosystems. That keeps data gravity on their side and simplifies auth, at the cost of vendor lock-in.

Pro-code agents

  • LLM provider SDKs or LLM gateways
  • Frameworks like LangChain, DSPy, Haystack

When more control is needed, e.g., excessive context engineering, niche integrations, custom logic to complement the agent, etc., pro-code agents are the way to go. They are the most flexible, but also the most complex to build and potentially maintain. The upfront cost is high, but the long-term cost can (often) be lower, especially when you scale.

Choosing the right tool

Start with a no-code or low-code builder to prototype, show progress quickly, and find product-market fit. As edge cases pile up and requirements become more complex, a pro-code rewrite might be necessary to optimize and scale.
